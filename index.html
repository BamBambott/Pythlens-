ENTROPY
On-chain verifiable randomness. Every random number generated by Pyth Entropy is cryptographically provable — meaning anyone can verify it wasn't manipulated. Essential for fair games, lotteries, NFT drops, and any application requiring trustless randomness on-chain.
// How it works
1. User commits to a random number
2. Pyth provider commits to a secret
3. Both values are revealed on-chain
4. The XOR of both = verifiable random output

Neither party can predict or manipulate the result. The demos below simulate this process.

// Confidence Interval Explained
Confidence interval = the ± band around each Pyth price, reflecting disagreement between publishers. A narrow band = high conviction. A wide band = market stress, outage, or liquidity divergence.

Stress score (bps) = (conf / price) × 10,000. Above 10bps = caution. Above 50bps = high stress.
